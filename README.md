# 🎮 Game Glitch Investigator: The Impossible Guesser

## 🚨 The Situation

You asked an AI to build a simple "Number Guessing Game" using Streamlit.
It wrote the code, ran away, and now the game is unplayable. 

- You can't win.
- The hints lie to you.
- The secret number seems to have commitment issues.

## 🛠️ Setup

1. Install dependencies: `pip install -r requirements.txt`
2. Run the broken app: `python -m streamlit run app.py`

## 🕵️‍♂️ Your Mission

1. **Play the game.** Open the "Developer Debug Info" tab in the app to see the secret number. Try to win.
2. **Find the State Bug.** Why does the secret number change every time you click "Submit"? Ask ChatGPT: *"How do I keep a variable from resetting in Streamlit when I click a button?"*
3. **Fix the Logic.** The hints ("Higher/Lower") are wrong. Fix them.
4. **Refactor & Test.** - Move the logic into `logic_utils.py`.
   - Run `pytest` in your terminal.
   - Keep fixing until all tests pass!

## 📝 Document Your Experience

- [ ] Describe the game's purpose.
The purpose of the game is to be able to guess a number. It was generated by AI and had a bunch of intentional bugs so I could practice debugging and refactoring code.
- [ ] Detail which bugs you found.
1. The hints were completely backwards. My input was 40, and it kept telling me to go higher even though the answer was 2.
  2. The game was glitching out when comparing numbers because it kept turning the secret number into a string randomly.
  3. The "New Game" button didn't work at all. After a game ended, I'd click it, but I was unable to submit any new guesses.
- [ ] Explain what fixes you applied.
 1. I refactored the code by moving all the core game logic out of `app.py` and into a separate `logic_utils.py` file to clean things up.
  2. I fixed the math comparison glitch by making sure both my guess and the secret number were always treated as integers using `int()`.
  3. I went into the code and swapped the hint text around so it actually points me in the right direction.
  4. I fixed the "New Game" button by making sure it resets the attempts, status, and history in the Streamlit session state.
  5. I wrote `pytest` cases to automatically test my logic and make sure the "Too High" and "Too Low" hints stay fixed.
